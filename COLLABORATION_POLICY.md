# Collaboration & Third-Party Policy

## Third-Party Engagement

### Authorized Collaborators

Only individuals explicitly added by SadderNowOddish as collaborators can:
- Commit to protected branches
- Approve pull requests
- Modify repository settings
- Manage releases

### Unauthorized Access

Any attempts to:
- Gain unauthorized repository access
- Bypass security controls
- Access without permission
- Misuse granted access

Will result in immediate removal and possible legal action.

## Case Studies & Research

### Requirement for Publication

Before conducting case studies, research, or analysis using Minni-gunns:

1. **Request Permission** via casestudies@minni-gunns.dev
2. **Provide**:
   - Research objectives
   - Publication details
   - Timeline
   - Attribution plan
   - Any data handling
3. **Await Approval** (14-day review)
4. **Obtain Agreement**: Sign terms if requested
5. **Include Attribution**: Must credit Minni-gunns prominently

### Unauthorized Case Studies

Using Minni-gunns without permission for:
- Academic publications
- Research papers
- Industry reports
- Benchmarking studies
- Security testing documentation
- Conference presentations

Constitutes copyright infringement and may result in:
- Cease and desist letter
- DMCA takedown notice
- Legal action
- Reputational consequences

## Derivative Works & Forks

### Acceptable Forks

You may fork Minni-gunns for:
- Personal development
- Internal organizational use
- Contributing improvements back

**Requirements**:
- Maintain copyright attribution
- Include LICENSE file unchanged
- Do not claim original authorship
- Link back to original repository

### Unacceptable Use

Forks may NOT be used to:
- Publish as your original work
- Remove attribution or copyright
- Compete commercially without license
- Hide modifications from upstream
- Conduct unauthorized case studies

## Third-Party Code Integration

### Before Adding Dependencies

1. Verify license compatibility
2. Check for security issues
3. Assess maintenance status
4. Document attribution
5. Get approval (if major change)

### License Compliance

Permitted licenses:
- ✅ MIT
- ✅ Apache 2.0
- ✅ BSD
- ✅ ISC
- ✅ GPL v3 (with review)

Not permitted:
- ❌ Proprietary/commercial
- ❌ AGPL without approval
- ❌ Restrictive licenses

## Collaboration Agreements

### Formal Partnerships

For significant collaborations:
1. Written agreement required
2. IP ownership clarified
3. Contribution terms defined
4. Attribution arrangements
5. Dispute resolution process

### MOU (Memorandum of Understanding)

Organizations using Minni-gunns in production should consider:
- Defining data handling
- Security responsibilities
- IP arrangements
- Support expectations
- Term and renewal

## Enforcement

### Monitoring

Minni-gunns maintains active monitoring for:
- Unauthorized forks claiming original work
- Unpermitted case studies
- IP violations
- Attribution removal

### Actions Taken

Upon discovery of violations:
1. Contact violator
2. Demand compliance
3. DMCA takedown if necessary
4. Pursue legal remedies
